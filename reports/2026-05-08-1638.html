03이탈 지표 한계 및 고도화 방향

현재 이탈 지표는 발송 패턴 기반의 1차 진단 수준입니다. 이탈 원인과 병원 맥락을 함께 파악할 수 있도록 아래 방향으로 고도화가 필요합니다. 우선순위 단기는 현재 보유 데이터로 적용 가능, 중기는 추가 수집 필요, 장기는 외부 데이터 연계가 요구됩니다.

발송량 증감률 단기
현재
조회 기간 내 총합·채널별 발송량 증감률 (%)
한계
발송량이 원래 적은 병원과 많은 병원이 동일한 기준으로 비교됨. 예를 들어, 월 10건 발송하던 병원이 5건 줄어도 -50%로 표시되어 월 1,000건 병원의 -50%와 같은 위험도로 판정될 수 있음
개선 방향
단순 증감률 대신 병원 규모(환자 수)를 고려한 상대적 발송률로 비교하거나, 절대 발송량을 함께 표시하여 실제 영향도를 파악할 수 있도록 개선 필요
미발송 일수 중기
현재
마지막 발송일로부터 경과 일수
한계
마지막 발송일 기준으로만 판단하기 때문에, 명절·시스템 점검으로 인한 일시적 중단인지 실제로 사용을 멈춘 것인지 구분이 안 됨
개선 방향
"며칠째 안 보냈다"는 정보 외에, 이전에도 같은 패턴이 반복됐는지, 지난 기간 대비 얼마나 더 길어졌는지를 함께 보면 실제 이탈 여부를 더 정확하게 판단할 수 있음
판정불가 병원 (27.7%) 단기
현재
운영 기간 부족 또는 데이터 부재 시 판정불가 처리
한계
판정불가 병원이 27.7%로 상당한 비중을 차지하며, 신규 병원(데이터 없음)과 발송량 부족(30건 미만)으로 원인은 구분되지만 운영 기간별 필터링이 없어 기간에 따른 현황 파악이 어려움
개선 방향
운영 기간별 필터(예: 신규 3개월 이하 / 기존 병원) 추가 → 신규 병원과 기존 병원을 나눠서 볼 수 있게 하면 판정불가 병원의 실질적인 관리 방향을 잡기 쉬워짐
이탈 원인 데이터 장기
현재
발송 패턴 지표만으로 위험도 판정
한계
발송량이 줄었다는 것은 알 수 있지만, 왜 줄었는지는 알 수 없음. 비용 문제인지, 사용이 불편해서인지, 효과를 못 느껴서인지 원인을 파악하기 어려워 병원마다 맞는 대응을 하기가 힘듦
개선 방향
CS 상담 기록이나 VOC 데이터를 연계하여 이탈 원인을 유형별로 분류 → "이 병원은 비용 문제, 저 병원은 사용 불편"처럼 원인에 맞는 대응이 가능해짐
